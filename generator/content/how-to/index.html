This page lists resources that are meant to help you get stuff done easily in the WebSpecs project. It is being expanded regularly.

Easily find out what is the best place at which to discuss a specific aspect that you have in mind.
Creating WebSpecs
You want to jump in and actually write some text, either from scratch or as a modification to someone else’s work. The details are on this page.
Bikeshed — Markdown for WebSpecs
WebSpecs can be written using a Markdown flavour known as “Bikeshed” that happens to have a lot of tricks up its sleeve that comes in handy when producing a specification. There’s a lot that can be known about Bikeshed, but you don’t need to learn more than the very basics in order to get started — and the rest ought to come easy.
ReSpec — HTML for WebSpecs
If you prefer HTML and JS to Markdown, WebSpecs can also be written using ReSpec. Like Bikeshed it takes care of a lot of the things you don’t want to have to think about. You can start from a very basic example and only tweak the configuration (near the top of the source) to set specStatus to "webspec" and repository to the user/repo-name string that fits your specification. Call the file index.src.html and you’re good to go!
